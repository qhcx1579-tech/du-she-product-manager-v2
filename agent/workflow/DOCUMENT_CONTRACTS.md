# 文档契约

## artifacts/SPEC.md

必须包含：

- 产品一句话定义
- 目标用户
- 用户场景
- 核心问题
- 功能范围
- 非功能要求
- 验收标准
- 明确不做什么
- 风险和待确认问题

## artifacts/BRIEF.md

必须包含：

- 设计目标
- 情绪关键词翻译后的具体方向
- 色彩方向
- 字体气质
- 信息密度
- 布局原则
- 组件风格
- 交互基调
- 禁止事项

## artifacts/DESIGN.md

必须包含：

- 页面清单
- 用户路径
- 视觉系统
- 关键页面结构
- 状态设计
- 响应式要求
- 可交付原型说明

## artifacts/PLAN.md

必须包含：

- Phase 列表
- 每个 Phase 的目标
- 每个 Phase 的可运行结果
- 每个 Phase 的验收标准
- 任务拆分
- 测试策略
- 审核点

## artifacts/BUILD_LOG.md

必须包含：

- 已完成任务
- 变更文件
- 运行方式
- 测试结果
- 审核记录
- 未完成事项

## artifacts/REVIEW.md

必须包含：

- 审核范围
- 发现的问题
- 严重级别
- 文件和位置
- 建议修复
- 是否通过

## artifacts/RELEASE.md

必须包含：

- 发布内容
- 隐私审计结果
- 排除上传清单
- 打包命令
- GitHub 上传步骤
- 发布后检查项

## artifacts/RUN_STATE.md

必须包含：

- 当前目标
- 当前阶段
- 下一个 agent
- 上一个 agent
- 交接次数
- 审核/修复尝试次数
- 失败次数
- 用户能力模式
- 停止条件
- 是否需要用户确认
- 下一步允许动作
